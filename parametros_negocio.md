## Estructura propuesta de la tabla

Esta tabla se agrega al `ruta_postgres.sql` en el siguiente turno.

```sql
CREATE TABLE ruta.client_parameters (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY,
  client_id BIGINT NOT NULL,
  parameter_key TEXT NOT NULL,
  parameter_value TEXT NOT NULL,
  value_type TEXT NOT NULL
    CHECK (value_type IN ('INTEGER', 'DECIMAL', 'STRING', 'BOOLEAN', 'JSON')),
  description TEXT,
  is_overrideable_by_client BOOLEAN NOT NULL DEFAULT TRUE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

  PRIMARY KEY (id, client_id),
  UNIQUE (client_id, parameter_key),
  CONSTRAINT fk_cparam_client FOREIGN KEY (client_id) REFERENCES ruta.clients(id)
) PARTITION BY LIST (client_id);
```

`is_overrideable_by_client = FALSE` significa que el parÃ¡metro es
solo configurable a nivel plataforma (ADMIN_RUTA), nunca por el
ADMIN_CLIENT. Ejemplo: lÃ­mite mÃ¡ximo de reintentos de webhook.
